Simple Object Orientation 4

Modified the app from Simple Object Orientation 3 to explore the flexibility introduced by dependency injection.

GenericPet, Cat and Dog still inherit from Pet abstract base class.

However, there are now more types of Cat and Dog. Each type of cat is a Cat and each type of dog is a Dog.

So it has avoided introducing a sub-class of Cat or Dog and instead the specific instances can be composed at runtime.

  • Enter a name and if you are creating a generic pet enter a species.
  • Then create the pet - either generic, dog or cat
  • Select the pet you want to interact with using the radio buttons
  • Generic pets use the makeNoise method from the Pet class
  • All types of cat are created using a Cat class, so they can all purr.
  • All types of dog are created using a Dog class, so they can all fetch.
  • All pets are created with a makeNoise function which is injected at run time.
  • This allows re-use of makeNoise functions across pets, e.g. generic pets and cats may both be able to hiss, although no shared noises have yet been implemented.
  • (Only) Cats have a purr method
  • (Only) Dogs have a fetch method
