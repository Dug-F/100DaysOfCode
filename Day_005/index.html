Simple Object Orientation 5

Refactor of the app from Simple Object Orientation 4 to replace radio buttons with select dropdowns.

The intention is to move towards implementing a factory pattern, but the code is not there yet.

GenericPet, Cat and Dog inherit from Pet abstract base class.

The makeNoise function is injected and allows the noise to be specified at run time.

From a user point of view the app is similar to yesterday, but now has dropdowns to select the pet rather than radio buttons.

  • Enter a name and if you are creating a generic pet enter a species.
  • Select the type of pet to create and then click the 'Create Pet' button.
  • The pets are of different types - either generic, dog or cat
  • Select the pet you want to interact with using the 'select a pet' select drowdown
  • Generic pets use the makeNoise method from the Pet class
  • Generic Pets, Cats and Dogs are created with a makeNoise function which is injected at run time
  • (Only) Cats have a purr method
  • (Only) Dogs have a fetch method
